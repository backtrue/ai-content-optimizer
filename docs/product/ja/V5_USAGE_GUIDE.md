# v5 適応型ハイブリッドスコアリング - 使用ガイド

## クイックスタート

### 1. コンテンツ分析の提出

#### ステップ 1：分析ページにアクセス
AI コンテンツオプティマイザーの分析ページにアクセスします。

#### ステップ 2：コンテンツを貼り付け
- プレーンテキストまたは HTML 形式をサポート
- 推奨：300 文字以上
- システムが自動的に形式を検出

#### ステップ 3：ターゲットキーワードを入力
- 最大 5 個のキーワード
- カンマで区切る
- 例：「ルテイン、眼の健康、サプリメント」

#### ステップ 4：メールアドレスを入力
- 完了時に結果リンクが送信されます
- メールアドレスが正しいことを確認してください

#### ステップ 5：送信
「分析を送信」ボタンをクリックします。コンテンツが分析キューに入ります。

### 2. 結果を待つ

#### 予想時間
- 分析は通常 **1～5 分**
- ピーク時は 10～15 分かかる場合があります

#### 進捗を確認
- システムがタスク ID を表示
- このタスク ID を保存して後で参照できます

#### メールを受信
- 完了時にメールが届きます
- メールに完全な結果リンクが含まれています
- リンクをクリックして詳細な分析を表示

## スコアを理解する

### デュアルトラックスコアリングシステム

v5 モデルは 2 つの独立したスコアリング次元を使用します：

#### 構造スコア（40% の重み）
コンテンツの**組織と表現**を測定

**HTML コンテンツの場合**
- 見出し構造（H1/H2 階層）
- 段落、リスト、表
- 可読性（文の長さ）
- 引用とデータ

**プレーンテキストコンテンツの場合**
- コンテンツの長さ
- 段落構造
- 可読性
- 引用とデータ

**改善提案**
- リストまたは表を追加
- 長い段落を分割
- 引用とデータを含める

#### 戦略スコア（60% の重み）
コンテンツの**説得力と完全性**を測定

**WHY - 問題定義（1～10 点）**
- 読者の痛点を明確に説明していますか？
- 「読者がこの記事を必要とする理由」を説明していますか？

**HOW - 実装方法（1～10 点）**
- ソリューションの原理を説明していますか？
- 具体的なステップまたはガイダンスを提供していますか？

**WHAT - ソリューション（1～10 点）**
- ソリューションを明確に要約していますか？
- 読者への実際の利益を強調していますか？

**改善提案**
- 問題説明の具体性を強化
- より多くの実践例を追加
- ソリューションを明確に要約

### 総合スコアの解釈

```
総合スコア = 構造スコア × 0.4 + 戦略スコア × 0.6
```

**スコア範囲**
- **80～100**：🌟 優秀 - 優れたコンテンツ品質、AI 引用の可能性
- **60～79**：👍 良好 - 基礎がしっかり、最適化の余地あり
- **40～59**：⚠️ 中程度 - 改善が必要、推奨事項に従う
- **0～39**：❌ 改善が必要 - コンテンツ戦略の見直しを検討

## よくある質問

### Q：プレーンテキストのスコアが HTML より低いのはなぜですか？
**A：** プレーンテキストモードは HTML タグ（H1/H2 など）をチェックできません。推奨事項：
- コンテンツを追加（1000 文字以上）
- 段落構造を改善
- 引用とデータを含める

### Q：WHY スコアが低い場合はどうすればよいですか？
**A：** 記事に問題説明が不足しています。推奨事項：
- 冒頭で読者が直面する痛点を明確に指摘
- 具体的な例で問題の重要性を説明
- 「読者がこの記事を必要とする理由」を説明

### Q：HOW スコアが低いことは何を意味しますか？
**A：** コンテンツに具体的な実行ガイダンスが不足しています。推奨事項：
- ステップバイステップの説明を追加
- 実践例または操作ガイドを含める
- ソリューションの原理を説明

### Q：WHAT スコアが低い場合はどうすればよいですか？
**A：** 結論またはソリューションが不明確です。推奨事項：
- 結論でソリューションを明確に要約
- 読者への実際の利益を強調
- 曖昧または一般的すぎる結論を避ける

### Q：メールはどのくらいで届きますか？
**A：** 通常 1～5 分です。15 分以内に受け取らない場合：
- スパムフォルダを確認
- メールアドレスが正しいことを確認
- タスク ID を使用して手動で結果を確認

### Q：同じ記事を再度分析できますか？
**A：** はい。各送信により新しいタスク ID と分析結果が生成されます。

### Q：結果はどのくらい保存されますか？
**A：** 結果は 7 日間保存されます。後で参照するためにメールまたはスクリーンショットを保存することをお勧めします。

## ベストプラクティス

### 高スコアコンテンツの作成

#### 1. 明確な問題定義（WHY を改善）
```
❌ 悪い例：「この記事はルテインサプリメントを紹介します。」
✅ 良い例：「年齢とともに、多くの人が視力低下に直面しています。ルテインは眼を保護するのに役立つ重要な栄養素です。この記事では、最適なサプリメントを選ぶ方法をガイドします。」
```

#### 2. 具体的な実装ステップ（HOW を改善）
```
❌ 悪い例：「良いサプリメントを選んでください。」
✅ 良い例：「サプリメントを選ぶときは、以下の要因を考慮してください：
1. 用量：1 日 10～20mg 推奨
2. 形式：遊離形式はエステル化形式より吸収が速い
3. 認証：USP または NSF 認証を探す
4. 組み合わせ：ルテインとゼアキサンチンの組み合わせがより効果的」
```

#### 3. 明確な結論（WHAT を改善）
```
❌ 悪い例：「適切なサプリメントを選ぶことが重要です。」
✅ 良い例：「要するに、ルテインサプリメントを選ぶには、用量、形式、認証を考慮する必要があります。10～20mg の用量から始め、有名なブランドを選び、3～6 ヶ月間継続して効果を観察することをお勧めします。」
```

#### 4. 証拠を追加（構造スコアを改善）
- 研究データを引用：「2021 年の研究では...」
- 具体的な数字を提供：「視力コントラスト感度を 35% 改善」
- ケーススタディを列挙：「多くの消費者が報告...」

#### 5. 可読性を改善（構造スコアを改善）
- リストと表を使用
- 長い段落を分割（< 380 文字）
- 文構造を簡潔に（平均 15～25 単語）

## 技術詳細

### サポートされているコンテンツ形式
- **プレーンテキスト**：テキストを直接貼り付け
- **HTML**：タグ付きの Web コンテンツ
- **Markdown**：自動的にプレーンテキストに変換

### コンテンツの制限
- **最小長**：300 文字以上推奨
- **最大長**：無制限（最初の 8000 文字を分析）

### プライバシーとセキュリティ
- コンテンツは分析にのみ使用され、保存または共有されません
- メールアドレスは結果通知にのみ使用されます
- すべての通信は暗号化されて送信されます
